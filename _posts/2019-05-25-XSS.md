XSS 란 cross-site scripting 의 약자이다. 이는 가장 기초적인 취약점 공격 방법의 일종으로 악의적인 사용자가 공격하려는 사이트에 스크립트를 넣는 기법을 말한다.

공격에 성공하면 사이트에 접속한 사용자는 삽입된 코드를 실행하게 되며, 보통 의도치 않은 행동을 수행시키거나 쿠키나 세션 토큰 등의 민감한 정보를 탈취한다.

XSS란 자바스크립트를 사용하여 공격하는 경우가 많다. 공격 방법이 단순 하고 가장 기초적이지만 많은 웹 사이트들이 조치를 해두지 않기 때문에 공격을 받는 경우가 많다.

이 공격은 여러 사용자가 접근 가능한 게시판에 코드를 삽입하거나 메일과 같은 매체를 통해서도 전파된다. html을 사용하는 것이기 때문에 text-only 게시판 같은 곳에서는 xss가 발생할 일이 없다.

###<공격기법>

### 1. 스크립트 태그

-	스크립트 태그로 자바스크립트를 실행한다.

```
<<script>alert('attack');</script>>
```

### 2. 자바스크립트 링크

-	링크 태그로 자바스크립트를 실행한다.

```
<<a href="javascript:alert('XSS')">XSS</a>>
```

-	브라우저에서 about: 링크와 같이 , javascript:로 시작하는 링크는 스크립트를 실행시킨다.

###3. 이벤트 속성

-	이벤트 속성을 사용한다.

```
<<img src='~' onerror="alert('XSS')">>
```

-	이벤트 속성으로 스크립트 실행 가능
-	주로 on~ 처럼 되어있는 속성이 이벤트 속성
-	자주 사용되는 이벤트 속성으로는 onload, onerror, onclick 등이 있다.

###4. 블랙리스트 우회

-	알려지지 않은 태그와 속성 사용

```
<<ruby oncopy = "alert('XSS')">XSS</ruby>>
```

-	위의 기법들을 막는 사이트에 사용

###5. 내용 난독화

-	따옴표로 감싸는 문자열 사이에 공백 문자들을 넣고 html 인코드를 하여 난독화
-	일부 브라우저에서 javascript: 링크 사이에 공백 문자가 들어갈 수 있고, html 인코드를 해도 디코드 된 내용이 출력 된다는 점을 이용.

```
<<a href="&#x6A;&#x61;&#x76;&#x61;&#x73;&#xA;&#x63;&#x72;&#x69;&#x70;&#x74;&#xA;&#x3A;&#xA;&#x61;&#x6C;&#x65;&#x72;&#x74;&#xA;&#x28;&#x27;&#x58;&#x53;&#x53;&#x27;&#x29;">XSS</a>>
```
