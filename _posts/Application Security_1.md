### FTP (File Transfer Protocol)

#### 개요

인터넷에 연결된 시스템 간 파일을 전송하는 통신 프로토콜 중 하나

-	사용자는 FTP 클라이언트 프로그램을 이용하여 FTP 서버에 접속한 후 파일을 송수신한다.
-	내부적으로는 <span style="color:red">TCP 프로토콜</span> 사용
-	FTP 는 FTP 클라이언트 프로글램을 사용해서 TCP로 접속하고 접속 이후에 사용자 ID와 패스워드를 입력 받아 인증 수행

#### FTP 특징

-	포트 2개 사용 > user,pass, get 등의 FTP 명령을 FTP 서버에 전송하기 위한 <span style="color:red">명령포트(21번)</span>와 실제 파일을 업로드하거나 다운로드를 하기 위한 <span style="color:red">데이터 포트</span> 사용

-	명령 포트는 고정되어있지만 데이터 포트는 전송 모드에 따라 변하게 됨

> -	Active Mode 인 경우 데이터 포트 : 20 번
> -	Passive Mode 인 경우 : FTP 서버가 자신의 데이터 포트를 결정해서 FTP 클라이언트에게 서버 데이터 포트 번호를 보내 줌
> -	FTP 클라이언트의 데이터 포트는 FTP 클라이언트 자신이 결정

-	명령 채널과 데이터 전송 채널이 독립적으로 동작
-	클라이언트가 명령 채널을 통해 서버에게 파일 전송을 요구하면 서버는 데이터 전송 채널을 통해 데이터를 전송하는 방식으로 동작
-	서버의 명령 채널은 21번 포트를 사용하고 데이터 전송 채널은 20번 포트를 사용함
